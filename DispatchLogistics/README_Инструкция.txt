ЧАСТЬ 1: СОЗДАНИЕ БАЗЫ ДАННЫХ В SSMS 2019
=======================

1. Откройте SQL Server Management Studio 2019 (SSMS).

2. Подключитесь к вашему локальному серверу SQL Server:
   - Имя сервера: . (точка — для сервера по умолчанию)
   - Или: .\SQLEXPRESS (если установлена Express-версия)
   - Аутентификация: Windows Authentication

3. В меню выберите File → Open → File...

4. Откройте файл:
   h:\Viktor xyu\DispatchLogistics\SQL\CreateDatabase.sql

5. Убедитесь, что в выпадающем списке базы данных (вверху окна)
   выбрано "master".

6. Нажмите кнопку Execute (или F5).

7. В окне Messages вы должны увидеть:
   - "База данных DispatchLogisticsDB создана."
   - "Все таблицы созданы успешно."
   - "Тестовые данные добавлены успешно."

8. Обновите Object Explorer (кнопка Refresh), раскройте Databases и
   убедитесь, что DispatchLogisticsDB появилась.

9. Раскройте DispatchLogisticsDB → Tables и убедитесь, что все 10 таблиц
   на месте:
   - dbo.Users
   - dbo.Clients
   - dbo.Transport
   - dbo.Tariffs
   - dbo.GeoPoints
   - dbo.Distances
   - dbo.AdditionalServices
   - dbo.Orders
   - dbo.OrderServices
   - dbo.OrderStatusHistory

10. Проверьте данные — выполните SELECT * FROM Clients и убедитесь,
    что есть 5 тестовых клиентов.

ВАЖНО: Если у вас SQL Server Express с именованным экземпляром,
откройте файл App.config и измените строку подключения:
  Data Source=.\SQLEXPRESS;Initial Catalog=DispatchLogisticsDB;Integrated Security=True;

ЧАСТЬ 2: ЗАПУСК В VISUAL STUDIO 2019
=======================

1. Откройте Visual Studio 2019.

2. Выберите File → Open → Project/Solution...

3. Перейдите к файлу:
   h:\Viktor xyu\DispatchLogistics\DispatchLogistics.sln

4. Дождитесь загрузки проекта.

5. Убедитесь, что целевая платформа — .NET Framework 4.7.2:
   - Правой кнопкой на проекте → Properties
   - Application → Target framework → .NET Framework 4.7.2

6. Проверьте строку подключения в App.config:
   - Откройте App.config
   - Убедитесь, что Data Source соответствует вашему серверу
   - Для локального сервера по умолчанию: Data Source=.
   - Для Express: Data Source=.\SQLEXPRESS

7. Соберите проект:
   - Build → Build Solution (или Ctrl+Shift+B)
   - В окне Output должно быть: "Build succeeded"

8. Запустите приложение:
   - Debug → Start (или F5)
   - Или Debug → Start Without Debugging (Ctrl+F5)

ЧАСТЬ 6: АРХИТЕКТУРА
=======================

Проект построен по трёхуровневой архитектуре:

  Presentation Layer (Forms)
       ↕
  Business Logic Layer (Services)
       ↕
  Data Access Layer (DataAccess + DatabaseHelper)
       ↕
  SQL Server Database

Преимущества:
  • Разделение ответственности
  • Все SQL-запросы параметризованы (защита от SQL-инъекций)
  • Модели данных отделены от логики БД
  • Вспомогательные классы переиспользуемы
  • Единый стиль UI через UIStyleHelper

ЧАСТЬ 7: ФОРМУЛА РАСЧЁТА СТОИМОСТИ
=======================

BaseDistanceCost = DistanceKm × CostPerKm   (если CostPerKM задан)
IdleCost         = IdleHours × CostPerHour  (если CostPerHour задан)
WeightCost       = CargoWeight × CostPerTon (если CostPerTon задан)
FuelCost         = FuelSurcharge            (если задан)
ServicesCost     = Сумма всех доп. услуг
Subtotal         = BaseDistanceCost + IdleCost + WeightCost + FuelCost + ServicesCost
Total            = Subtotal × SeasonalCoefficient

Пример:
  Москва → Санкт-Петербург (710 км), тариф "Стандартный"
  Масса: 8 тонн, простой: 2 часа, доп. услуги: погрузка + разгрузка (6000 руб.)
  
  BaseDistanceCost = 710 × 35 = 24 850 руб.
  IdleCost         = 2 × 500 = 1 000 руб.
  WeightCost       = 8 × 200 = 1 600 руб.
  FuelCost         = 500 руб.
  ServicesCost     = 3 000 + 3 000 = 6 000 руб.
  Subtotal         = 24 850 + 1 000 + 1 600 + 500 + 6 000 = 33 950 руб.
  Total            = 33 950 × 1.00 = 33 950.00 руб.

ЧАСТЬ 8: ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ
=======================

Проблема 1: "Cannot connect to database"
  Решение: Проверьте строку подключения в App.config.
           Убедитесь, что SQL Server запущен.
           Для SQLEXPRESS: Data Source=.\SQLEXPRESS

Проблема 2: "Build failed — missing references"
  Решение: Убедитесь, что установлены:
           - System.Data.SqlClient (NuGet: Install-Package System.Data.SqlClient)
           - .NET Framework 4.7.2 или выше

Проблема 3: Таблицы пустые
  Решение: Перезапустите SQL-скрипт CreateDatabase.sql

Проблема 4: Форма заказа не считает стоимость
  Решение: Убедитесь, что выбран тариф с заполненными ставками.
           Проверьте, что расстояние > 0.
